# 代码同步与维护指南

本文档用于记录和指导如何将原作者的 `EveSDE_2.0` 更新同步至本地，并在不冲突的情况下推送到我们自己的代码库并进行二次开发。

## 当前远程仓库（Remote）配置

通过在终端中执行 `git remote -v` 可以查看到目前的远程仓库配置：

- **`upstream` (上游仓库)**: `https://github.com/EstamelGG/EveSDE_2.0.git`
  - 作用：只能用来**单向获取**原作者发布的新版本代码。
- **`origin` (自己的仓库)**: `https://github.com/AkaJianguo/EVE_SDE.git`
  - 作用：我们自己维护和备份的云端仓库。

## 分支架构策略（防止代码冲突）

为了能安全地修改代码且日后依然能一键同步原作者的更新，请严格遵守以下分支规范：

- **`main` 分支**：纯净分支，**绝对不要在这里写代码**。它的唯一目标就是与原作者保持一致。
- **`dev` 分支**：你的专属开发分支，所有的自定义修改、提交都在该分支上进行。

## 你的日常工作流（必看）

### 场景一：日常修改代码并提交
默认你平时敲代码或修改配置时，都要确保自己在 `dev` 分支上操作。如果你不在本地 `dev` 分支，请先在终端执行 `git checkout dev`。

1. **写代码**：在 `dev` 分支修改你想要的任何文件。
2. **提交与推送**：
   ```bash
   git add .
   git commit -m "你的修改描述"
   git push origin dev
   ```
*(或者你可以直接使用 VS Code 左侧源代码管理界面的“提交”和“同步”按钮)*

### 场景二：原作者发布了更新，你想同步

当原作者 (`EstamelGG`) 代码更新时，按下面 3 步走，可以完美、无冲突地把新特性合并到你的代码中：

**步骤 1：让纯净的 main 抓取更新代码**
```bash
git checkout main          # 切回主干
git pull upstream main     # 无损拉取原作者最新更新（绝不会冲突）
git push origin main       # 顺手把最新纯净版推送到自己的 GitHub
```

**步骤 2：把最新的更新融入你自己的代码中**
```bash
git checkout dev           # 切回你的开发分支
git merge main             # 拿主干的最新代码去合并 dev
```
*(注意：如果原作者更新的代码和你在 dev 里改过的代码恰好是同一个文件的同一行，这里可能会提示冲突，请在 VS Code 中点击“解决冲突”并在解决后保存并 commit。)*

**步骤 3：保存合并成果**
```bash
git push origin dev        # 把最终包含了原作者更新的 dev 分支推送到自己的云端
```
