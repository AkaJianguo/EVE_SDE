# 代码同步与维护指南

本文档用于记录和指导如何将原作者的 `EveSDE_2.0` 更新同步至本地，并将其推送到我们自己的代码库中。

## 当前远程仓库（Remote）配置情况

通过在终端中执行 `git remote -v` 可以查看到目前的远程仓库配置如下：

- **`upstream` (上游仓库)**: `https://github.com/EstamelGG/EveSDE_2.0.git`
  - 作用：用来获取原作者发布的新版本和代码。
- **`origin` (自己的仓库)**: `https://github.com/AkaJianguo/EVE_SDE.git`
  - 作用：我们自己维护和备份的云端仓库。

## 日常同步流程 (Sync)

当原作者（`EstamelGG/EveSDE_2.0`）有了新的代码提交时，请按照以下步骤将其同步到你自己的仓库：

### 1. 从原作者的上游仓库拉取最新代码

在项目根目录下打开终端，执行：

```bash
git pull upstream main
```

> **注意**：如果原作者有更新，这条命令会自动将最新代码合并到你当前的本地主分支 (`main`)。
> 若本地没有进行过修改，此过程应该没有任何冲突。如果发生冲突（通常是因为你也修改了本地代码），需要手动解决冲突后再 `commit`。

### 2. 将同步好的代码推送到属于自己的仓库

本地合并完成且测试无误后，将最新代码上传至你自己的 GitHub 仓库：

```bash
git push origin main
```

## 贡献和自定义修改的建议

如果你需要在这个项目上新增具有自己特色的功能或做大规模的结构修改，为了避免在未来合并原作者代码时产生严重冲突，建议按照以下分支管理方式进行：

1. **在此 `main` 分支上保持与原仓库的完全同步**（严格意义上不在该分支写自己的代码，仅用于同步获取更新）。
2. **新建一个自己的特性分支进行修改**, 如需要修改代码，使用：`git checkout -b my-features`。
3. 当上游（原作者）更新后：
   - 先以 `main` 进行同步：`git checkout main` -> `git pull upstream main`
   - 将本地修改应用到新版的 `main`：`git checkout my-features` -> `git rebase main` 或 `git merge main`。
   - 测试
